# CLOCKWORK 보상 체계 v0.1

> 상태: [제안]
> 목적: 메트로베니아식 탐사 보상과 필수 진행 보상을 분리하고, 맵 노드와 Unity 데이터가 같은 분류를 사용하도록 한다. 기존 정본 룰북은 수정하지 않으며, 본 문서는 검토 후 차기 정본에 병합한다.

`유틸리티 MOD`, `전투 MOD`, `고정 탐사 MOD`는 보상 역할을 구분하기 위한 임시 데이터 태그다. 개별 MOD 이름·효과·수치와 이 분류 체계 자체는 공식 확정이 아니다.

## 1. 보상 축

| 분류 | ID 예시 | 역할 | 주요 획득처 | 반복 획득 |
|---|---|---|---|---|
| 이동·전투 능력 | `ability_*` | 필수 경로와 조작 확장 | 메인 능력 보스 | 불가 |
| 진행 키 | `*_key`, `core_*`, `*_power` | 시설·액트 내부 잠금 해제 | 메인 진행·수문장 | 불가 |
| 무기 | `weapon_*` | 공격 방식 확장 | 스토리·선택 보스·대형 탐사 | 불가 |
| 개조 | `mod_*` | 장착 빌드 변화 | 보물방·선택 탐사 완주 | 불가 |
| MOD 슬롯 | `module_slot_plus_1` | 동시 장착 수 증가 | 액트 후반 고난도 탐사 | 불가 |
| 하트 코어 파편 | `heart_core_fragment` | 4개 완성 시 최대 체력 1칸 증가 | 짧은 샛길·수집 퍼즐 | 불가 |
| 렌티움 용량 파편 | `lentium_capacity_fragment` | 3개 완성 시 회복·자원 게이지 상한 1단계 증가 | 능력 심화방·외곽 탐사 | 불가 |
| 고철 부품 | `salvage_parts` | 모르비 제작·무기 강화에 쓰는 단일 반복 재화 | 적·상자·부품고 | 가능 |
| 기록 | `archive_record_*` | 세계관·시설 기능 설명. 기억 코어와 분리 | 환경 탐사·선택 지역 | 불가 |
| 이동 편의 | `*_shortcut`, `*_station` | 백트래킹 단축 | 루프 종점·승강기·터미널 | 불가 |

## 2. 핵심 규칙

1. 이동 능력은 메인 능력 보스만 지급한다. 선택 지역 보상으로 필수 진행을 열지 않는다.
2. 대형 선택 지역은 최초 완주 시 고유 영구 보상 1개와 입구 복귀 숏컷을 지급한다.
3. 고철 부품은 유일한 반복 재화다. 반복 파밍 없이도 메인 엔딩에 도달할 수 있어야 한다.
4. 일반 재화는 긴 선택 지역의 최종 보상이 될 수 없다. 고철 부품은 영구 보상에 붙는 부가 보상으로만 사용한다.
5. MOD는 완제품이며 무작위 옵션을 사용하지 않는다. MOD 3택1은 하나를 즉시 획득하고, 미선택 2개는 다음 액트부터 모르비 제작 목록에 등록한다.
6. MOD는 1칸에서 시작한다. ACT1 터미너스에서 2칸, ACT2 후반에서 최대 3칸으로 확장한다.
7. 기억 코어는 보스·스토리 전용이다. 일반 기록과 황금 결정 흔적은 기억 보상으로 계산하지 않는다.
8. 체크포인트, 승강기, 수문, 복귀 숏컷도 정식 보상으로 기록한다.

## 3. 보상 등급

| 등급 | 플레이 규모 | 주 보상 |
|---|---|---|
| `minor` | 30초 이내 샛길 | 고철 부품·기록·파편 |
| `standard` | 1~3분 보상실 | 완성 파편·고정 MOD·대량 고철 |
| `major` | 5~10분 선택 루프 | MOD 3택1·고유 MOD·슬롯·무기 |
| `progression` | 메인 보스·액트 전환 | 능력·기억·필수 통로 |

## 4. ACT1 배치안

| 위치 | 최초 클리어 보상 | 보상 의도 |
|---|---|---|
| 림부스 부품 더미 | 시작 MOD 3택1 | 수리 전에는 정체불명 부품으로 표시하고 모르비가 식별 |
| 구 폐기물 선별장 | 고정 탐사 MOD + 고철 부품 | 갈고리 재방문과 선별기 루프 완주 보상 |
| 공방 부품고 | 대량 고철 부품 | 짧은 소형 보상실 |
| 정수장치 취수구 | 렌티움 용량 파편 | 갈고리 첫 심화 사용 보상 |
| 폐쇄 송수로 | 유틸리티 MOD 3택1 | 수위·밸브 퍼즐 완주 보상 |
| 진흙 동굴 | 하트 코어 파편 + 정체불명 결정 흔적 | 생존 성장과 기억② 복선 분리 |
| 심층 굴착지 | 전투 MOD 3택1 | 더블점프 심화 지역 완주 보상 |
| 터미너스 상단 정비 선반 | MOD 슬롯 +1 | ACT1 빌드 용량 확장 |

ACT1 보상 예산은 이동 능력 2개, 기억 2개, MOD 획득 기회 4개, MOD 슬롯 1개, 하트 코어 파편 1개, 렌티움 용량 파편 1개로 시작한다. 파편은 ACT2 이후 탐사까지 이어지는 장기 수집축이다.

## 5. ACT2 외곽 배치안

| 위치 | 최초 클리어 보상 | 부가 보상 |
|---|---|---|
| 서부 외벽 화물 야적장·회수로 | 렌티움 용량 파편 | 화물 운송 기록 |
| 동부 외곽 재활용 처리장·황폐 가도 | 하트 코어 파편 | 재활용 처리 기록 |

두 외곽 지역은 벽타기 획득 후 동시에 열리지만 메인 진행에는 필요하지 않다. 각각 반대 종류의 파편을 배치해 어느 쪽을 먼저 탐사해도 영구 성장을 얻도록 한다.

## 6. 맵 데이터 규약

보상 노드는 다음 속성을 사용한다.

```html
<div data-reward="mod_choice_utility"
     data-reward-type="mod"
     data-reward-tier="major"
     data-first-clear-reward="mod_choice_utility"
     data-repeatable-reward="none"
     data-choice-group="act1_aqueduct">
</div>
```

- `data-reward`: 기존 호환용 대표 보상 ID
- `data-reward-type`: `ability`, `key`, `memory`, `weapon`, `mod`, `slot`, `fragment`, `currency`, `record`, `shortcut`
- `data-reward-tier`: `minor`, `standard`, `major`, `progression`
- `data-first-clear-reward`: 최초 클리어 때 지급되는 공백 구분 ID 목록
- `data-repeatable-reward`: 재방문 때 반복 획득 가능한 ID. 없으면 `none`
- `data-choice-group`: 3택1 등 상호 배타 선택 그룹
- `data-reward-state`: 식별 전·후처럼 서사 상태가 필요한 경우 사용

## 7. 아직 결정하지 않는 것

- 개별 MOD 이름과 수치
- 고철 부품의 정확한 드롭률과 강화 비용
- 무기 강화 단계 수
- 기록 수집 마일스톤의 총개수와 보상
- 파편 완성 연출과 HUD 형태
