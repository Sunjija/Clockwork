# CLOCKWORK 음악 문서

이 디렉터리는 음악 기획, Suno 생성 실험과 승인 상태를 다른 게임 문서 및 에셋 작업과 분리해 관리한다.

## 문서 구성

| 문서 | 역할 | 갱신 시점 |
| --- | --- | --- |
| [01_region_music_direction.md](01_region_music_direction.md) | ACT별·지역별 정서, 플레이 기능과 음악 방향 | 맵, 서사 또는 지역 기능이 변경될 때 |
| [02_suno_anchor_experiments.md](02_suno_anchor_experiments.md) | Suno 프롬프트, 생성 ID, 청취 피드백과 기각 사유의 시간순 기록 | 생성 또는 청취 평가 직후 |
| [03_act1_suno_production_plan.md](03_act1_suno_production_plan.md) | ACT1 곡 목록, 제작 순서, 변주 관계와 미완료 작업 | ACT1 제작 범위나 우선순위가 변경될 때 |
| [04_clockwork_music_production_principles.md](04_clockwork_music_production_principles.md) | 전 작품에 공통 적용할 작곡·편성·루프·검수 원칙 | 반복 실험에서 재사용할 원칙이 확인될 때 |

## 상태 규칙

| 상태 | 의미 |
| --- | --- |
| 제안 | 문서로 설계했지만 아직 생성하거나 검증하지 않음 |
| 청취 대기 | 후보가 생성됐지만 사용자가 평가하지 않음 |
| 비교 후보 | 장점 일부를 후속 실험에 참고할 수 있음 |
| 기각 | 정본과 후속 시드로 사용하지 않음. 실패 원인은 보존 |
| 승인 후보 | 사용자가 방향을 승인했으며 편집·루프 검증이 남음 |
| 정본 | 게임 적용 대상으로 확정되고 원본 음원과 사용 범위가 지정됨 |

현재 정본으로 확정된 음악은 없다. Suno 링크와 생성 ID는 실험 이력이며, 사용자 승인 없이 지역곡·메인 테마 또는 모티프의 정본으로 간주하지 않는다.

## 작업 흐름

1. `04`의 공통 원칙과 `01`의 지역 기능을 확인한다.
2. `03`에서 이번 제작 대상과 앞뒤 곡의 변주 관계를 정한다.
3. 한 프롬프트에서 후보 두 개를 생성한다.
4. 결과와 청취 피드백을 `02`에 기록한다.
5. 승인 후보만 루프, 음량, 전환과 게임 적용 가능성을 검증한다.
6. 최종 승인 후 정본 상태와 원본 음원 위치를 이 문서에 추가한다.

## 저장 원칙

- 문서와 프롬프트는 `docs/music`에 저장한다.
- 실제 음원 파일은 저작권과 저장 용량을 확인한 뒤 별도 에셋 경로를 정한다.
- 기각 후보를 삭제해 실험을 반복하지 않도록 기각 사유는 유지한다.
- 음악 변경은 가능하면 다른 코드·맵·캐릭터 에셋 변경과 분리해 커밋한다.
