# 티크 행동 애니메이션 치트시트

## 공통 제작 규칙

- 모든 프레임은 발바닥 중앙을 같은 기준점에 둔다.
- 가로와 세로를 따로 늘리지 않는다. 불투명 캐릭터 높이만 102px로 정규화한다.
- 머리, 몸통, 팔, 다리의 내부 비율은 승인된 3/4 원본을 유지한다.
- 조작 판정은 즉시 시작하고, 짧은 준비 동작은 판정을 늦추지 않는 시각 레이어로 처리한다.
- 한 이미지에는 한 동작의 한 키포즈만 둔다. 런타임에서 여러 포즈가 들어간 시트를 직접 잘라 쓰지 않는다.
- 접지 프레임에서는 발 미끄러짐이 없도록 발 위치와 캐릭터 이동량을 함께 검사한다.
- 걷기 재생 위치는 경과 시간이 아니라 실제 지상 이동거리에 동기화한다.
- 스프라이트에는 보간 필터를 사용하지 않고 정수 화면 좌표에 배치한다.

## 행동별 키포즈

| 행동 | 키포즈 순서 | 런타임 적용 |
|---|---|---|
| 대기 | 정착 → 압축 → 상승 → 복귀 | 1.60초 반복, 발 고정, 정착과 복귀를 길게 유지 |
| 걷기 | 접지 → 다운 → 통과 → 업 → 반대발 접지 → 다운 → 통과 → 업 | 0.72초 반복, 접지를 길게 두고 업 포즈를 짧게 통과, 양팔과 양다리는 반대로 교차 |
| 점프 | 압축 → 이륙 접지 → 상승 → 정점 → 낙하 → 착지 압축 | 입력 즉시 상승, 0.14초 이륙 표현, 속도 기반 공중 포즈, 0.18초 착지 |
| 더블점프 | 압축 → 추진 → 신전 → 복귀 | 공중 판정은 즉시 적용, 0.34초 시각 시퀀스 |
| 대시 | 준비 → 추진 → 활성 → 회복 | 이동은 즉시 시작, 0.32초, 활성 구간에만 잔상 강화 |
| 공격 | 준비 → 실행 → 팔로스루 → 회복 | 0.42초, 24~68% 구간만 공격 판정/궤적 표시 |
| 피격 | 충격 → 반동 → 회복 | 대미지 시스템 구현 시 연결 |
| 다운 | 충격 → 붕괴 → 정착 → 기상 | 체력/부활 시스템 구현 시 연결 |

## 동작별 외부 기준과 티크 적용

아래의 `외부 기준`은 자료에 명시된 사실이고, `티크 적용`은 CLOCKWORK용 설계 결정이다. 자료를 학습시킨 것이 아니라 키포즈와 타이밍을 제작·검수 기준으로 사용한다.

| 동작 | 외부 기준 | 티크 적용 |
|---|---|---|
| 걷기 | 접지, 다운, 통과, 업의 네 핵심 포즈가 양발 기준으로 반복된다. | 8프레임은 유지하되 0.50초 균등 재생을 폐기하고 0.72초 비균등 재생으로 변경한다. 손상된 85kg 몸체를 고려해 접지를 길게 잡는다. |
| 점프·착지 | 중립, 예비 압축, 이륙 접지, 공중, 낙하 접지, 착지 압축, 정착으로 나누며 이륙과 착지 접지 포즈가 중요하다. | 입력과 물리 상승은 즉시 시작한다. 압축 이미지는 캐릭터가 이미 움직이는 동안 겹쳐 보여 조작 지연을 만들지 않는다. 정점은 세로 속도로 선택한다. |
| 더블점프 | 새 추진력을 보여 주는 압축과 신전, 이후 공중 포즈로의 복귀가 필요하다. | 첫 점프와 구분되는 코어 발광 및 추진 포즈를 사용하고, 추진 이후 복귀 프레임을 가장 길게 둔다. |
| 대시 | 게임 동작은 반응성을 위해 긴 사전 동작을 피하고 준비, 추진/활성, 회복의 대비를 명확히 한다. | 이동 판정은 첫 프레임부터 시작한다. 준비는 30ms만 보이고 활성 실루엣을 가장 오래 유지한다. |
| 공격 | 기본 자세, 예비, 타격, 팔로스루, 회복의 실루엣이 명확해야 하며 공격·회복 타이밍은 플레이 규칙과 일치해야 한다. | 타격까지 120ms, 타격 포즈 40ms, 회복 200ms로 배분한다. 공격 판정은 기존 24~68% 구간을 유지한다. |
| 피격 | 충격 방향, 최대 반동 포즈, 무게 흡수, 복귀가 즉시 읽혀야 한다. | 피격 시스템 구현 시 좌우 방향별 반동 포즈와 짧은 히트스톱을 별도 클립으로 만든다. 현재는 사양만 예약한다. |

## 프레임 체류시간

| 클립 | 프레임별 체류시간(ms) | 의도 |
|---|---|---|
| 걷기 | 130, 90, 80, 60, 150, 90, 70, 50 | 접지 유지, 빠른 업 통과, 미세한 비대칭 보행 |
| 점프 이륙 | 59, 81 | 입력 즉시 이동하면서 압축에서 이륙 접지로 전환 |
| 더블점프 | 50, 70, 100, 120 | 추진은 짧고 공중 자세 복귀는 길게 |
| 대시 | 30, 60, 130, 100 | 준비를 최소화하고 활성 실루엣과 회복을 읽히게 함 |
| 공격 | 70, 50, 40, 60, 90, 110 | 타격까지 빠르게, 충격 직후 회수와 복귀에 무게 부여 |

이 수치는 외부 자료의 고정값이 아니라 티크의 체급과 손상 상태에 맞춘 1차 튜닝값이다. 실제 플레이 테스트 후 변경할 수 있다.

## 런타임 동기화 규칙

- 걷기 한 주기는 지상 이동거리 155px에 대응한다. 정상 최고 이동속도 250px/s에서는 약 0.62초, 초반 파손 상태 215px/s에서는 약 0.72초 주기가 된다.
- 가속·감속 중에는 이동한 거리만큼만 걷기 프레임이 진행된다. 벽에 막히면 걷기 포즈도 진행되지 않는다.
- 정지할 때는 가까운 양발 접지 포즈로 걷기 위상을 정렬한 뒤 대기로 전환한다.
- 점프, 더블점프, 대시, 공격의 입력과 물리 판정은 첫 프레임에 시작한다. 시각적 예비동작은 판정을 지연시키지 않는다.
- 각 클립은 `timeline`의 누적 종료 비율을 사용해 프레임마다 서로 다른 체류시간을 갖는다.

## 검수 체크

1. 모든 프레임의 불투명 높이가 같은가.
2. 같은 행동의 발바닥 기준선이 같은가.
3. 머리 폭과 높이가 프레임 사이에서 변하지 않는가.
4. 몸통 대비 팔·다리 길이가 변하지 않는가.
5. 접지한 발이 화면에서 미끄러지지 않는가.
6. 준비 동작 때문에 실제 입력 반응이 늦어지지 않는가.
7. 공격의 시각 궤적과 실제 활성 판정 구간이 일치하는가.
8. 점프 상승·정점·하강·착지가 속도 변화와 일치하는가.

## v3 이미지 검수 결과

- 런타임 참조 경로를 `generated/v3/<동작>/`으로 통일해 대기, 걷기, 점프, 더블점프, 대시, 공격을 서로 독립된 클립으로 유지한다.
- 걷기 8프레임은 각 키포즈를 개별 이미지로 다시 제작했다. 전반 4프레임과 후반 4프레임에서 앞팔·뒷팔과 앞다리·뒷다리가 반대로 교차한다.
- 대시 4프레임은 준비, 추진, 활성, 회복 실루엣이 겹치지 않도록 새 이미지로 교체했다.
- 대기, 점프, 더블점프, 공격의 신규 생성 후보는 머리·몸통·팔다리 비율 허용치를 벗어나 최종본에서 제외했다. 이 클립들은 기존 승인 프레임을 v3 폴더에 복제해 동작별 파일 구조만 통일했다.
- 모든 최종 프레임은 640×512 투명 캔버스, 소스 기준점 `(320, 480)`, 발 기준선 `y=480`을 사용한다.
- 최종 비교표는 `prototypes/lento-vertical-slice/assets/tique/generated/v3/preview-all-actions-v3.png`, 동작별 반복 프리뷰는 같은 폴더의 `previews/`에 둔다.
- 인게임 표시 배율은 모든 동작에서 `0.34`로 고정한다. 크기가 다른 이미지를 배율로 보정하지 않고 `tique-character-model-lock.md`에 맞는 동일 체형 이미지만 사용한다.
- 걷기·공격·대시의 황동색과 코어색은 대기 클립의 평균 팔레트에 맞춘다. 색 보정은 따뜻한 금속색과 시안 발광부에만 적용하고 검은 외곽선과 투명도는 변경하지 않는다.

## 인게임 모델 고정 규칙

- 인게임용 티크에는 등 뒤 태엽 키와 돌출된 태엽 장치를 넣지 않는다.
- 원본 캐릭터 시트는 비율과 색상 참고용으로 사용하되, 태엽은 생성 프롬프트와 최종 에셋에서 제외한다.
