# Suno Workflow

## Create

1. Open Suno Create and use the current highest-quality model available to the account unless the user requests another model.
2. Use Advanced mode when separate title, style, exclusions, or inspiration controls are needed.
3. Enable Instrumental before generation. Verify the active state instead of assuming an empty lyric field is sufficient.
4. Set the title to the exact region, boss, or event name.
5. Put positive musical behavior in Styles and concrete failure modes in Exclude styles.
6. Generate two variants.
7. Wait until both entries have playable durations before reporting completion.

## Browser Safety and Ownership

- A request to generate music authorizes creation and credit use for the requested scope.
- Do not publish, share, like, dislike, delete, rename existing unrelated tracks, or alter account settings without explicit instruction.
- Do not enter passwords or authentication codes. Hand control to the user for authentication.
- Do not claim to have listened when only DOM metadata, duration, or title was inspected.

## Compare

Compare variants using the user's listening feedback or an available audio-analysis workflow. Check:

- motif clarity
- regional identity
- fatigue over repetition
- loop or edit point
- phase readability for bosses
- unwanted vocals or genre drift
- continuity with the preceding and following cue

When the user selects a preferred variant, record its Suno link or ID as the reference candidate. Do not call it canonical until the user explicitly approves it.

## Iterate

Change one major dimension per revision:

- melody and motif treatment
- pulse or meter
- instrumentation roles
- arrangement density
- acoustic or production space
- phase transition behavior

Keep the successful dimensions fixed. Use Inspo only from a user-selected or explicitly delegated source track. Avoid generating many near-identical variants without first identifying what failed.
